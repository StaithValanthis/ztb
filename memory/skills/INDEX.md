# Skills catalogue

Reusable know-how the firm reads back instead of re-deriving. Two kinds only:
**process** (how we do a recurring engineering/ops task) and **fact** (a durable, verified reference).
A skill is **NEVER** a trading strategy or market "edge" — edges are plugins that must pass the
evidence gate (cost-aware backtest → OOS → ≥3-week forward-test). Keep this catalogue small and concise.
